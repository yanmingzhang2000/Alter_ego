# Alter Ego

### 一、 核心设计理念

**“不看流水账，只看红绿灯。”**

看板的定位是“业务体检仪”，数据必须服务于决策。针对底层数据多系统割裂的现状，MVP 版本在展示上采用“双轨制”：以财务（Galaxy）数据作为业绩定音锤，以业务系统（CRM\+BPMS）数据作为过程归因抓手，并在前端做好差异标识。

### 二、 看板信息架构 \(IA\) 与模块划分

MVP 版本建议采用单页面“三段式”布局，从宏观到微观层层递进。

### 三、 针对“灵魂拷问”的核心模块设计

#### 应对拷问一：SKA \(大客户\) 护城河模块

负责人最怕悄无声息的流失。该模块将客户全生命周期指标做到极简，直接把大客户的异动拍在负责人脸上。

- **大客户水位线：** 固定展示年度签约额 TOP 客户的当期“签约金额”与“财务确认收入”，直观反映大客户盘子是否稳固。

- **沉默大客户预警（高亮）：** 锁定“连续 90 天无新订单”的客户池。一旦有大客户进入该池，直接标红，要求对应工作室负责人介入。

- **单客毛利透视：** 展示大客户的“单客服务成本”与“单客毛利”。避免出现“客户营收大，但全靠堆人力、买流量，实际没赚钱”的虚假繁荣。

#### 应对拷问二：人效 X 光机模块

针对人均绩效下滑，该模块负责拆解是“人多了”（分母变大）还是“钱少了”（分子变小），甚至是“分工比例错了”。

- **多口径人效趋势图：** 同步折线展示“人均营收”、“人均毛利”与“人均净利”，直观判断业务增长是否带来了真实利润。

- **中台与职能成本摊薄视角：** 按 MVP 简化规则，展示按营收占比分摊后的“事业部中台成本”及“公共固定成本”。如果某工作室人效暴跌，可迅速查明是被公摊成本拖累，还是自身直接人力成本失控。

- **前端与交付配比分析：** 对比展示“仅 BD 销售岗人数”与“实际交付在岗人数（包含跨工作室支援）”的产出比。排查是否出现“销售签单猛但交付跟不上”，或“交付闲置导致人效拉垮”。

#### 应对拷问三：对账公信力与利润真伪模块

确保负责人对看板数据有 100% 的信任度，避免陷入“对数据”的内部扯皮。

- **数据置信度标签：** 凡是涉及跨系统提取的数据，一侧展示金额，一侧展示红绿灯标签。绿色代表 CRM 订单额与 Galaxy 财务入账匹配；黄色代表存在差额，提示需人工线下核对。

- **跨工作室支援台账：** 仅做标签与汇总展示，不影响核心毛利公式。用一个独立的视图展示“本月各工作室对外支援规模”，让负责人在定性考核时有据可依，安抚被白嫖人力的团队。

### 四、 Agent MVP 落地与数据闭环 \(二期铺垫\)

按照你的规划，MVP 阶段暂不开发复杂的 AI 对话或模拟测算，但我们需要为二期 Agent 的“智能归因”储备结构化数据。

- **规则化预警推送：** 设定硬性阈值（如：单项目毛利率 \< 20%）。每日数据 T\+1 更新后，系统自动向负责人飞书/企微推送《每日毛利异常及流失风险简报》。

- **“人工判决”收集（核心预埋）：** 在所有被看板判定为“异常”（如 SKA 掉量、人效不达标）的数据条目后，增加一个必填的下拉选择框（如：客情变动、投放成本激增、跨部门支援导致）与文本备注框。

- **月度快照固化：** 每月 5 号自动打包当月的所有基础数据、异常记录及负责人的备注，生成一份不可篡改的《月度经营快照》。这份快照结合上述的“人工判决”，将成为后续训练 Agent 自动生成“业务归因报告”的最优质语料库。
